# #0-2. Custom plugin

몇가지 도움이 되는 커스텀 플러그인을 설치하고 시작하겠습니다.



### ComfyUI crystools

1. ComfyUI 상단에는 Comfy Manager가 설치되어 접근할 수 있는 버튼이 있습니다. Manager 버튼을 클릭합니다.

<figure><img src="../.gitbook/assets/image (78).png" alt=""><figcaption></figcaption></figure>



2. Comfy Manager를 통해 Custom Node 다운로드, Model 다운로드, ComfyUI 업데이트 등 ComfyUI 사용에 필수적인 기능들이 포함되어 있습니다.

<figure><img src="../.gitbook/assets/image (79).png" alt=""><figcaption></figcaption></figure>



3. 상단의 Custom Nodes Manager를 클릭합니다.

<figure><img src="../.gitbook/assets/image (80).png" alt=""><figcaption></figcaption></figure>



4. Custom Nodes Manager 상단 검색란에 crystools를 검색하면 ComfyUI-Crystools를 확인할 수 있습니다. 해당 노드를 Install 버튼을 클릭하여 설치합니다.

<figure><img src="../.gitbook/assets/image (81).png" alt=""><figcaption></figcaption></figure>



5. 설치 후 다시 세션을 열면, 우상단에 리소스 모니터, 진행률 표시줄 및 경과 시간, 메타데이터 확인할 수 있도록 플러그인이 추가됩니다.

<figure><img src="../.gitbook/assets/image (82).png" alt=""><figcaption></figcaption></figure>



### Model Manager



ComfyUI Manager에 포함되어 있는 model manager는 comfyui repository에서 관리하는 모델들을 검색하고 다운로드 할 수 있도록 되어 있습니다.&#x20;

하지만 실제 모델 다운로드 및 활용 시 Huggingface / CivitAI 등에서 checkpoint를 다운로드 하는 경우가 발생하게 되는데, 이러한 경우에 유용하게 활용할 수 있는 플러그인으로 ComfyUI-Model-Manager가 있습니다.



1.  ComfyUI 상단에는 Comfy Manager가 설치되어 접근할 수 있는 버튼이 있습니다. Manager 버튼을 클릭합니다.

    <figure><img src="../.gitbook/assets/image (78).png" alt=""><figcaption></figcaption></figure>


2. Comfy Manager를 통해 Custom Node 다운로드, Model 다운로드, ComfyUI 업데이트 등 ComfyUI 사용에 필수적인 기능들이 포함되어 있습니다.

<figure><img src="../.gitbook/assets/image (79).png" alt=""><figcaption></figcaption></figure>



3. 상단의 Custom Nodes Manager를 클릭합니다.

<figure><img src="../.gitbook/assets/image (80).png" alt=""><figcaption></figcaption></figure>



4. Custom Nodes Manager 상단 검색란에 model manager를 검색하면 ComfyUI-Model Manager를 확인할 수 있습니다. 해당 노드를 Install 버튼을 클릭하여 설치합니다.

<figure><img src="../.gitbook/assets/image (83).png" alt=""><figcaption></figcaption></figure>



5. 설치 후 restart 하면 우측 상단에 Model Manager가 설치된 것을 확인할 수 있습니다.

<figure><img src="../.gitbook/assets/image (84).png" alt=""><figcaption></figcaption></figure>



6. Model Manager는 Huggingface / CivitAI 접근에 API 키를 사용합니다. 좌측 상단의 메뉴(C 마크)를 클릭하고, 설정에 들어갑니다.

<figure><img src="../.gitbook/assets/image (85).png" alt=""><figcaption></figcaption></figure>



7. Model Manager 항목을 클릭하면 API 키를 입력할 수 있습니다. CivitAI와 Huggingface API 키를 받아 입력합니다.

```
civitai : https://civitai.com/
huggingface : https://huggingface.co/
```



8. 이후 model manager를 클릭하여 다운로드 버튼을 클릭합니다.

<figure><img src="../.gitbook/assets/image (86).png" alt=""><figcaption></figcaption></figure>



9. Create Download Task를 클릭합니다.

<figure><img src="../.gitbook/assets/image (87).png" alt=""><figcaption></figcaption></figure>



10. civitai / huggingface의 링크를 입력하면, 아래와 같이 모델들을 다운로드할 수 있습니다.

```
https://civitai.com/models/124421/xxmix9realisticsdxl?modelVersionId=163192
```

<figure><img src="../.gitbook/assets/image (88).png" alt=""><figcaption></figcaption></figure>
