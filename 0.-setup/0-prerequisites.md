## ComfyUI란?

ComfyUI는 Stable Diffusion을 비롯한 다양한 생성형 AI 모델을 **노드 기반 인터페이스**로 제어할 수 있는 오픈소스 추론 엔진입니다.

### 🧩 쉬운 비유: 레고 블록처럼 조립하는 AI

ComfyUI를 **레고 블록**에 비유해 볼 수 있습니다:
- **노드(Node)** = 레고 블록 하나하나 (각각의 기능을 담당)
- **연결선(Edge)** = 블록들을 연결하는 끼워 맞춤 부분
- **워크플로우(Workflow)** = 완성된 레고 작품 (전체 이미지 생성 과정)

기존의 WebUI(A1111, Forge 등)와 달리 ComfyUI는 각 처리 단계를 독립적인 **노드**로 분리하여 사용자가 직접 파이프라인을 설계할 수 있습니다. 이를 통해:

- 모델 조합의 자유도가 높고
- 워크플로우를 시각적으로 이해하고 공유할 수 있으며
- 복잡한 생성 파이프라인을 코드 없이 구성할 수 있습니다

> **초보자 팁**: ComfyUI는 프로그래밍을 전혀 모르더라도 사용할 수 있습니다. 마치 파워포인트에서 도형을 연결하듯이, 마우스로 노드를 드래그하고 연결하기만 하면 됩니다!

## 핵심 용어 정리

### Checkpoint (체크포인트)

체크포인트는 이미지 생성에 사용되는 **메인 모델 파일**입니다. Stable Diffusion의 모든 구성 요소(U-Net, VAE, Text Encoder)가 하나의 파일에 통합되어 있습니다.

**🎨 쉬운 비유**: Checkpoint는 **AI 화가의 두뇌**입니다. 어떤 화가(모델)를 선택하느냐에 따라 그림 스타일이 완전히 달라집니다.

- **SD 1.5**: 약 2~4GB, 512x512 해상도에 최적화
- **SDXL**: 약 6~7GB, 1024x1024 해상도 지원
- **파일 형식**: `.safetensors` (권장) 또는 `.ckpt`
- **저장 경로**: `ComfyUI/models/checkpoints/`

대표적인 체크포인트:
- `sd_xl_base_1.0.safetensors` (SDXL 공식 모델)
- `dreamshaper_8.safetensors` (SD1.5 커뮤니티 모델)

> **초보자 팁**: 체크포인트는 용량이 크므로 다운로드에 시간이 걸립니다. 워크샵에서는 미리 준비된 모델을 사용합니다.

### VAE (Variational Autoencoder)

VAE는 **이미지와 잠재 공간(Latent Space) 간의 변환**을 담당하는 모델입니다.

**🔄 쉬운 비유**: VAE는 **통역사**입니다. AI가 이해할 수 있는 언어(잠재 공간)와 우리가 볼 수 있는 이미지(픽셀) 사이를 번역해줍니다.

- **Encoder**: 이미지(픽셀)를 잠재 공간으로 압축
- **Decoder**: 잠재 공간의 데이터를 다시 이미지로 복원

대부분의 체크포인트에 VAE가 포함되어 있지만, 별도의 VAE를 사용하면 색감이나 디테일이 개선될 수 있습니다.

- **저장 경로**: `ComfyUI/models/vae/`

### LoRA (Low-Rank Adaptation)

LoRA는 기존 체크포인트 모델을 **적은 파라미터로 미세 조정**한 경량 모델입니다.

**🎭 쉬운 비유**: LoRA는 **스타일 필터**입니다. 인스타그램 필터처럼 원본 모델(체크포인트)은 그대로 두고, 특정 스타일만 덧입힐 수 있습니다.

- 체크포인트를 교체하지 않고 특정 스타일이나 캐릭터를 추가할 수 있음
- 파일 크기가 작아 (수십~수백 MB) 여러 개를 조합(Stacking)할 수 있음
- **저장 경로**: `ComfyUI/models/loras/`

예시: 애니메이션 스타일 LoRA, 특정 인물 LoRA, LCM LoRA (가속화) 등

> **초보자 팁**: LoRA는 여러 개를 동시에 사용할 수 있습니다. 예를 들어 "애니메이션 스타일 LoRA" + "특정 캐릭터 LoRA"를 함께 적용할 수 있습니다!

### Latent (잠재 공간)

Latent은 이미지가 **압축된 수학적 표현 공간**입니다.

**🏭 쉬운 비유**: Latent는 **AI의 작업실**입니다. 우리 눈에는 보이지 않지만, AI는 이 작업실에서 그림을 그리다가 완성되면 최종 이미지로 변환해서 보여줍니다.

Stable Diffusion은 픽셀 공간이 아닌 잠재 공간에서 이미지를 생성합니다. 이를 통해:

- 계산량을 크게 줄일 수 있음 (512x512 이미지 → 64x64 잠재 벡터)
- 의미론적으로 더 풍부한 표현이 가능

ComfyUI에서 `Empty Latent Image` 노드는 초기 노이즈를 생성하고, `VAE Decode` 노드가 최종 잠재 벡터를 이미지로 변환합니다.

> **초보자 팁**: Latent 공간에서 작업하는 것이 왜 중요할까요? 마치 큰 캔버스(고해상도 이미지)에 직접 그리는 대신, 작은 스케치(Latent)로 먼저 작업해서 시간과 자원을 절약할 수 있기 때문입니다.

### 기타 핵심 용어

| 용어 | 설명 | 쉬운 비유 |
|------|------|-----------|
| **CLIP** | 텍스트 프롬프트를 모델이 이해할 수 있는 임베딩으로 변환하는 텍스트 인코더 | 📝 번역기: 한국어를 AI가 이해하는 언어로 번역 |
| **KSampler** | 노이즈 제거 과정(디노이징)을 수행하는 샘플러 노드 | 🖌️ 지우개: 흐릿한 그림에서 점점 선명한 그림으로 다듬음 |
| **CFG Scale** | 프롬프트에 대한 모델의 따름 정도 (높을수록 프롬프트에 충실) | 🎯 충실도 조절기: 7~11이 적당, 너무 높으면 과장됨 |
| **Steps** | 노이즈 제거 반복 횟수 (높을수록 정교하지만 느림) | ⏱️ 다듬기 횟수: 20~30번이면 충분, 많을수록 시간 오래 걸림 |
| **Seed** | 초기 노이즈의 랜덤 시드값 (같은 시드 = 같은 결과) | 🎲 주사위 번호: 같은 번호면 항상 같은 결과 |
| **Scheduler** | 각 스텝에서의 노이즈 감소 패턴을 정의 | 📊 작업 스케줄: 어떤 순서로 디테일을 다듬을지 결정 |
| **Positive Prompt** | 생성하고자 하는 이미지에 대한 설명 | ✅ 포함할 것: "이런 걸 그려주세요" |
| **Negative Prompt** | 생성에서 제외하고 싶은 요소에 대한 설명 | ❌ 제외할 것: "이건 그리지 마세요" |

## ComfyUI 인터페이스 구성

ComfyUI의 기본 인터페이스는 다음과 같이 구성됩니다:

1. **캔버스**: 노드를 배치하고 연결하는 작업 공간
2. **노드**: 각 처리 단계를 나타내는 블록 (더블클릭으로 검색/추가)
3. **엣지(연결선)**: 노드 간의 데이터 흐름을 나타내는 선 (색상별로 데이터 타입 구분)
4. **사이드바**: 워크플로우 관리, 큐 관리
5. **하단 패널**: 실행 로그 및 진행 상황

### 노드의 기본 구조

```
┌─────────────────────────┐
│   노드 이름 (예: KSampler) │
├─────────────────────────┤
│ 입력 ●                   │  ← 왼쪽: 입력 포트 (다른 노드에서 데이터 받음)
│                          │
│   [설정 옵션들]           │  ← 중앙: 노드의 설정값
│                          │
│                   ● 출력  │  ← 오른쪽: 출력 포트 (다른 노드로 데이터 전달)
└─────────────────────────┘
```

> **초보자 팁**: 노드를 레고 블록처럼 생각하세요. 왼쪽의 구멍(입력)에 다른 블록의 돌기(출력)를 끼워 넣으면 연결됩니다!

### 노드 색상 규칙

ComfyUI에서 노드의 입출력 포트는 **색상으로 데이터 타입을 구분**합니다:

| 색상 | 데이터 타입 | 무엇을 의미하나요? |
|------|-------------|-------------------|
| 보라색 | MODEL (모델) | AI 화가의 두뇌 |
| 노란색 | CLIP (텍스트 인코더) | 프롬프트 번역기 |
| 빨간색 | VAE | 이미지 변환기 |
| 주황색 | CONDITIONING (조건) | 번역된 프롬프트 지시사항 |
| 분홍색 | LATENT (잠재 벡터) | AI의 작업실 데이터 |
| 초록색 | IMAGE (이미지) | 최종 완성 이미지 |

> **초보자 팁**: 같은 색상의 출력 포트와 입력 포트를 연결하면 됩니다. 마치 전선을 색깔별로 연결하는 것과 같습니다!

### 기본 마우스 조작법

| 조작 | 동작 |
|------|------|
| **마우스 드래그** | 캔버스 이동 (손바닥으로 화면 밀기) |
| **마우스 휠** | 확대/축소 (줌 인/아웃) |
| **더블 클릭** | 노드 검색 창 열기 |
| **Ctrl + Enter** (또는 Cmd + Enter) | 워크플로우 실행 |
| **노드 위에서 드래그** | 노드 이동 |
| **출력 포트 → 입력 포트 드래그** | 노드 연결 |
| **연결선 클릭 후 Delete** | 연결 삭제 |

> **초보자 팁**: 실수로 노드를 삭제했다면? Ctrl+Z (또는 Cmd+Z)로 되돌리기가 가능합니다!

## 사전 설정

워크샵을 원활하게 진행하기 위해 아래 사항을 확인하세요:

1. **AWS 계정**: 워크샵용 AWS 계정이 준비되어 있어야 합니다
2. **브라우저**: Chrome 또는 Firefox 최신 버전 권장
3. **Hugging Face 계정**: 일부 모델 다운로드 시 필요 (https://huggingface.co)
4. **CivitAI 계정**: 커뮤니티 모델 다운로드 시 필요 (https://civitai.com)

> **초보자 팁**: Hugging Face와 CivitAI는 AI 모델을 공유하는 커뮤니티입니다. 무료로 가입할 수 있으며, 수많은 체크포인트와 LoRA를 다운로드할 수 있습니다.
